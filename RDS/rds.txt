Single-AZ DB instance deployment (1 instance) : Creates a single DB instance without standby instances. This setup provides: 99.5% uptime and No data redundancy..  (20$/Month)

Multi-AZ DB instance deployment (2 instances) : Creates a primary DB instance with a non-readable standby instance in a separate Availability Zone. 
This setup provides: 99.95% uptime.. Redundancy across Availability Zones (40$/Month)

Multi-AZ DB cluster deployment (3 instances): Creates a primary DB instance with two readable standbys in separate Availability Zones. This setup provides 99.95% uptime.. Redundancy across Availability Zones.. Increased read capacity.. Reduced write latency (60$/Month)

Leader Node : Plan the query execution

Compute Node : Stores the actual data.

WLM : Workload management : AutoWLM
WLM lets you create queues - Each and every queue gets a dedicated slice of memory and concurrency slots.

AutoWLM : Redshift automatically manage the memory and concurrency
Manual WLM : You have to define queues, memory and concurrency slots.

Queury Queue - A holding lane for queries. 
concurrency slots - How many queries can run simultaneously in a queue.
Memory - How much cluster memory that queue gets.
Short Query Acceleration (SQA) - Automaticallt fast track the shortqueries, so they dont wait behind big queries.

DMS : Database Migration Service

1. RDS : Source Database  : schema with some info
2. RDS : Target Database  : plan schema

3. Create a DMS cluster.. THis act as a intermidiate component.. 

4. Create Source endpoint and test connectivity to DMS

5. Create Target endpoint and test connectivity to DMS

6. Create a DMS task and configure to replicate the data.. 


Full load : Replicate all existing data once.

CDC : Change data capture : Ongoing replications

Full load + cdc : Once all changes and then continue with replicating ongoing chnages.. 

Homogeneous Migration : Same class source and target db migration (mysql --> mysql)
Heterogeneous Migration : Diff class source and target db migration (db2 --> post, mongo --> postgre/mysql)
--> For hetro, We need to use SCT (Schema convertion tool)
