Lambda supports 3 Invocation Models:

Synchronous Invocation: The caller send a request for response, lambda runs the function and retrun the result directly to the caller. If the lmabda fucntion fails, The caller should retry, lambda wont retry automatically.
--> API Gateway, Lamnbda function, ALB..

Asynchronous Invocation: The caller sends a request for response, lamdba responds with 202 status code.. Lambda rus the function and retur the result to the caller. If the lambda function fails, it is going to send the event to DLQ (Dead Letter Queue). Also, it retries 2 times.
--> SQS, SNS, EventBridge

Event Source Mapping : Lambda polls the source for meessages/records and invokes the lambda when it finds any records / batc of records.. 
--> Apache kafka, Kenins Data streams, SQS..

Lambda function has 3 life cycle phases:

1. INIT Phase : Initilisation Phase : Download the code, Initilize the runtime env and then it runs the code
2. INVOKE Phase : It executes the handler function. THis is actual phase where our code or business logics runs.. 
3. SHUTDOWN Phase : After a priod of time, It goes inactive, IN Backend AWS Destroy the environment and free the resources..


cold start : INIT + INVOKE : First invoke after a long time will take high latency.. ex: using a car after long time takes time to start the engine..

Warm Start : INVOKE Phase :  Subsequent invocations / Freq invocations keeps the env alive.. Low latency.. Ex: using a cart daily, makes the start quicker.. 

How can you reduse the cold start period time/duration..?? 
--> Provisioned cuncurrancy
--> SnapStart **


Lambda SnapStart : Resumes frozen VM snapshots from a fast cache.
Provisioned Concurrency : Keeps a specified number of environments constantly warm.
