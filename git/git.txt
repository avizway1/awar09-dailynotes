Branching : Branching helps us to create a seperate line of development for features, bug fixes or experimental changes ---> Without touching our main codebase.

git branch			--> tells us in which branch we are.


TBD : Trunk Based Development : its a git branching strategy, we directly dont work on Main/master branches, We always create a seperate branch based on the code change purposes.. Once the feature is fully implemnted and tested, We merge this branch into main branch, then we can delete this branch. 


git branch feature-x			--> THis creates a branch with name "feature-x"

git checkout feature-x			--> switch to branch (Traditional command)

Goldenrule / Thumb rule : never rebase a branch that has been pushed to a shared/public repo.

Rebase changes the commit hashes, it destroys teammenbers base hisory, It can multiply the duplicate commits. 

Local-only branches: rebase as much as you want to, as this cleans up the history before anyone else can see it..
Central/Shared Repo: always use git merge.. Even use git pull to get the team mebers work, before pushing your changes..
