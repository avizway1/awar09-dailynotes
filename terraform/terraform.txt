Resource Block: We can define what need to create in AWS account by using the tf code. 

Resource block contians 3 components..

resource-type		: aws_instance	(has to be same as defined in tf documentation)
Name for block 		: webserver / dbserver	(you can give anything, but no duplications with in the project)
Arguments			: ami id, instance type, security group..

terraform fmt				--> Run this as first thing to get proper allignment.
terraform validate			--> helps to identify syntax and config errors in tf file.

terraform plan 				--> Its a dryrun, tells us what happens if we apply this code. 
terraform apply				--> Create the resources in AWS env. 

terraform apply	--auto-approve		--> Non interactive apply. wont ask for confirmation. 

terraform destroy			--> Deletes the resources from our environment.

If any changes happened outside of the terraform, We can identify, We call it as "drift detection"..

1. Just Detect			: terraform plan -refresh-only
2. Accept the drift		: terraform apply -refresh-only (when someone made valid change - update statefile)
3. Revert the drift		: terraform apply				(our code is truth - undo the manual change)

Terraform logs

ERROR	: Errors only
WARN	: Warning + Error
INFO	: General operatinal info + Warning + error
DEBUG	: Api calls + General operatinal info + Warning + error
TRACE	: Everything - Every API call, response, Decision + General operatinal info + Warning + error

--> No collabarations with other team members.
--> No locking feature.

  backend "local" {
    path = "/Users/avizway/Desktop/Batch 9/tflocal/terraform.tfstate"
  }
}

we need to migrate the statefile using below command

terraform init -migrate-state

---

We always store the "tfstate file" inside an "S3 bucket".
1. Enable versioning on s3 bucket.
2. Enable s3 bucket object lock with "compliance mode", So no one can delete the file, Even if the root user try to delete, if fails till the retention day, we configure.


terraform {
  backend "s3" {
    bucket = "aviz.tf.state.myproject"
	key = "projectx/terraform.tfstate"
	region = "ap-south-1"
	use_lockfile = true
  }
}


*Rare cases, we do this
terraform force-unlock <lockid>
