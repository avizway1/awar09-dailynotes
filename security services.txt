Amazon SQS : Simple Queue Service : 

1. Standard
2. FIFO : 


Standard : At-least-once delivery, message ordering isn't preserved
--> At-least once delivery
--> Best-effort ordering

FIFO : First-in-first-out delivery, message ordering is preserved
--> First-in-first-out delivery
--> Exactly-once processing

DLQ / Dead-letter Queue : A dedicated Queue designed to store undelivered messages 

Max Message retention period : 14 Days

--> Decoupling microservices required this SQS service
--> for traffic smoothing and ratelimiting

SQS 

Poll : The consumer actively asks the provider for new data at regular intervals.
Push : The provider automatically sends data to the consumer exactly moment it become available.
